1. The Principle of Mathematical Induction

Why Induction Works: The Domino Effect

Imagine an infinite line of dominoes. How can you prove they'll all fall?

üéØ Two Key Ingredients:

  1. Base Case: The first domino falls
  2. Inductive Step: If any domino falls, the next one falls too

Conclusion: ALL dominoes will fall! This is the essence of mathematical induction.

Formal Statement

To prove a statement P(n) is true for all natural numbers n ‚â• n‚ÇÄ:

1

Base Case: Prove P(n‚ÇÄ) is true

2

Inductive Hypothesis: Assume P(k) is true for some k ‚â• n‚ÇÄ

3

Inductive Step: Prove P(k+1) is true using the assumption

‚à¥

Conclusion: P(n) is true for all n ‚â• n‚ÇÄ

2. Anatomy of an Inductive Proof

Template for Weak Induction

Theorem: For all n ‚â• n‚ÇÄ, P(n) is true

Proof by induction:

  BASE CASE (n = n‚ÇÄ):
    [Verify P(n‚ÇÄ) is true directly]

  INDUCTIVE HYPOTHESIS:
    Assume P(k) is true for some arbitrary k ‚â• n‚ÇÄ
    That is, assume [state what P(k) means]

  INDUCTIVE STEP:
    We must prove P(k+1)
    [Use the assumption P(k) to prove P(k+1)]

    Starting with P(k+1):
    [Mathematical manipulation]
    [Apply inductive hypothesis]
    [Continue until proven]

  CONCLUSION:
    By mathematical induction, P(n) is true for all n ‚â• n‚ÇÄ. ‚àé

Critical Components

  • Base Case: Usually straightforward; verify by direct calculation
  • Inductive Hypothesis (IH): The assumption that links k to k+1
  • Inductive Step: The creative part; must explicitly use the IH

3. Classic Example: Sum Formula

Theorem: Sum of First n Integers

1 + 2 + 3 + ... + n = n(n+1)/2

Base

n = 1:
LHS = 1
RHS = 1(1+1)/2 = 1
LHS = RHS ‚úì

IH

Inductive Hypothesis:
Assume for n = k:
1 + 2 + 3 + ... + k = k(k+1)/2

Step

Prove for n = k+1:
We need to show: 1 + 2 + ... + k + (k+1) = (k+1)(k+2)/2

Work

LHS = [1 + 2 + ... + k] + (k+1)
    = k(k+1)/2 + (k+1) (by IH)
    = k(k+1)/2 + 2(k+1)/2
    = [k(k+1) + 2(k+1)]/2
    = (k+1)(k+2)/2
    = RHS ‚úì

‚àé

Conclusion: By mathematical induction, the formula holds for all n ‚â• 1.

5. Strong Induction

Weak vs Strong Induction

Weak Induction:

To prove P(k+1), assume only P(k)

Strong Induction:

To prove P(k+1), assume P(1), P(2), ..., P(k) are ALL true

When to use Strong Induction:
  • When P(k+1) depends on multiple previous cases
  • Recursive definitions with multiple base cases (like Fibonacci)
  • When you're not sure which previous case you'll need

Example: Fundamental Theorem of Arithmetic

Theorem: Every integer n ‚â• 2 can be written as a product of primes

Base

n = 2: 2 is prime ‚úì

IH

Strong Hypothesis: Assume all integers from 2 to k can be written as products of primes

Step

Consider k+1:
Case 1: If k+1 is prime, we're done ‚úì
Case 2: If k+1 is composite, then k+1 = a¬∑b where 2 ‚â§ a,b ‚â§ k
  By IH, both a and b are products of primes
  Therefore k+1 = a¬∑b is also a product of primes ‚úì

üíª Prime Factorization

def prime_factorization(n):
    """Find all prime factors of n (mirrors strong induction)"""
    if n < 2:
        return []

    factors = []
    divisor = 2

    while divisor * divisor <= n:
        while n % divisor == 0:
            factors.append(divisor)
            n //= divisor
        divisor += 1

    if n > 1:
        factors.append(n)

    return factors

# Test
for n in [12, 30, 100, 97, 2023]:
    factors = prime_factorization(n)
    product = 1
    for f in factors:
        product *= f
    print(f"{n} = {' √ó '.join(map(str, factors))} (check: {product})")

6. Well-Ordering Principle

The Well-Ordering Principle (WOP)

Every non-empty set of non-negative integers has a least element.

Connection to Induction

Mathematical induction and the Well-Ordering Principle are logically equivalent!

Proof by WOP Template:

  1. Assume the theorem is false
  2. Let S = {counterexamples}
  3. By WOP, S has a smallest element n‚ÇÄ
  4. Show n‚ÇÄ can't actually be smallest ‚Üí contradiction

Example: Division Algorithm via WOP

Theorem: For any integers a,b with b > 0, there exist unique q,r such that a = bq + r with 0 ‚â§ r < b

1

Consider S = {a - bk : k ‚àà ‚Ñ§, a - bk ‚â• 0}

2

S is non-empty (take k large negative) and contains only non-negative integers

3

By WOP, S has a least element r = a - bq for some q

4

Must show r < b: If r ‚â• b, then a - b(q+1) = r - b ‚â• 0, contradicting minimality of r

7. Common Mistakes in Induction Proofs

‚ö†Ô∏è Mistake 1: Missing or Wrong Base Case

Claim: All horses are the same color (FALSE!)

Faulty "Proof": Base case n=1: One horse is one color ‚úì
IH: Any k horses are the same color
Step: Given k+1 horses, remove first horse ‚Üí k horses same color (IH)
Remove last horse ‚Üí k horses same color (IH)
Therefore all k+1 horses same color...

Error: The step from n=1 to n=2 fails! With 2 horses, removing one leaves no overlap to connect them.

‚ö†Ô∏è Mistake 2: Not Using the Inductive Hypothesis

BAD: Proving P(k+1) without referencing P(k)

This is just a direct proof, not induction!

GOOD: Explicitly show where IH is used

... = [expression with k] + (k+1)
    = k(k+1)/2 + (k+1)  ‚Üê BY IH
    = ...

‚ö†Ô∏è Mistake 3: Circular Reasoning

Circular: Assuming what you're trying to prove

In the inductive step for P(k+1), you cannot assume P(k+1) is true! Only assume P(k) (or P(1)...P(k) for strong induction).

‚ö†Ô∏è Mistake 4: Wrong Direction

Wrong: Starting with what you want to prove
P(k+1) = ... = P(k) which is true

Right: Starting with P(k) (known by IH)
P(k) = ... = P(k+1) which is what we wanted

8. Applications of Induction

Algorithm Correctness

Induction proves that algorithms work for all input sizes

# Prove: Binary search finds target in sorted array (or returns -1)

def binary_search(arr, target, left, right):
    """
    Correctness by strong induction on array size (right - left + 1)

    Base: Size 0 (left > right) ‚Üí returns -1 ‚úì
    Base: Size 1 (left == right) ‚Üí checks arr[left] ‚úì

    IH: Assume binary_search works for arrays of size < k

    Step: For size k:
      - Split into two halves, each size < k
      - If target in left half, recurse on left (works by IH)
      - If target in right half, recurse on right (works by IH)
      - Otherwise target not present ‚úì
    """
    if left > right:
        return -1

    mid = (left + right) // 2

    if arr[mid] == target:
        return mid
    elif arr[mid] > target:
        return binary_search(arr, target, left, mid - 1)
    else:
        return binary_search(arr, target, mid + 1, right)

Data Structure Properties

Binary Tree Node Count

Theorem: A complete binary tree of height h has 2^(h+1) - 1 nodes

class TreeNode:
    def __init__(self, val=0):
        self.val = val
        self.left = None
        self.right = None

def count_nodes(root):
    """
    Proof by structural induction:
    Base: Empty tree (height -1) has 0 = 2^0 - 1 nodes ‚úì
    Base: Single node (height 0) has 1 = 2^1 - 1 nodes ‚úì

    IH: Trees of height < h satisfy the formula

    Step: Tree of height h has:
      - 1 root
      - Left subtree of height h-1
      - Right subtree of height h-1

      Total = 1 + (2^h - 1) + (2^h - 1)  [by IH]
            = 1 + 2¬∑2^h - 2
            = 2^(h+1) - 1 ‚úì
    """
    if not root:
        return 0
    return 1 + count_nodes(root.left) + count_nodes(root.right)

9. Induction in Python: Loop Invariants

Loop Invariants = Induction

A loop invariant is a property that's true before and after each loop iteration

Structure mirrors induction:

  • Initialization (Base): Invariant true before loop starts
  • Maintenance (Inductive Step): If true before iteration i, true after
  • Termination (Conclusion): When loop ends, invariant gives correctness

üíª Example: Insertion Sort Correctness

def insertion_sort(arr):
    """
    Loop Invariant: At start of iteration i, arr[0:i] is sorted

    Initialization: Before i=1, arr[0:1] has one element ‚Üí sorted ‚úì

    Maintenance: Assume arr[0:i] sorted
      - Insert arr[i] into correct position in arr[0:i]
      - Now arr[0:i+1] is sorted ‚úì

    Termination: When i=n, arr[0:n] is sorted (entire array) ‚úì
    """
    n = len(arr)

    for i in range(1, n):
        # INVARIANT: arr[0:i] is sorted
        key = arr[i]
        j = i - 1

        # Insert key into sorted portion
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
        # INVARIANT: arr[0:i+1] is sorted

    # INVARIANT: arr[0:n] is sorted
    return arr

# Test
test_arr = [5, 2, 9, 1, 5, 6]
print(f"Original: {test_arr}")
print(f"Sorted: {insertion_sort(test_arr.copy())}")

üíª Fibonacci with Induction

def fibonacci(n):
    """
    Compute nth Fibonacci number
    Correctness by strong induction

    Base: n=0 ‚Üí return 0 ‚úì
    Base: n=1 ‚Üí return 1 ‚úì

    IH: fibonacci(k) correct for all k < n

    Step: For n ‚â• 2:
      fibonacci(n) = fibonacci(n-1) + fibonacci(n-2)
      Both calls correct by IH ‚úì
    """
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)

# Iterative version with loop invariant
def fibonacci_iter(n):
    """
    Loop Invariant: After iteration i:
      a = F(i), b = F(i+1)
    """
    if n <= 1:
        return n

    a, b = 0, 1  # F(0)=0, F(1)=1
    for i in range(2, n + 1):
        # INVARIANT: a=F(i-2), b=F(i-1)
        a, b = b, a + b
        # INVARIANT: a=F(i-1), b=F(i)

    return b

# Verify both
for n in range(10):
    rec = fibonacci(n)
    itr = fibonacci_iter(n)
    assert rec == itr
    print(f"F({n}) = {rec}")
