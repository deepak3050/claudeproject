## USAGE NOTES
- **Sections 1-3:** Always include (mandatory for any project)
- **Section 4:** Include only if technical specs matter (apps, code, data projects)
- **Section 5:** Include if tone/style is important (writing, design, content)
- **Section 6:** Include examples for complex logic or when accuracy matters
- **Section 7:** Include only if you want to prevent common mistakes

**Total tokens:** ~200-400 words (vs 1000+ for verbose specs)
